== Installation

NOTE: Only tested on Emacs version 25+.

The configuration is maintained using https://github.com/raxod502/straight.el[straight.el^] for package management and https://github.com/jwiegley/use-package[use-package^] for package initialization and configuration.

Everything is self-contained in `init.el` and other elisp files. The package manager is _bootstraped_ if needed. Simply clone and launch emacs:

[source,shell]
----
mv -v ~/.emacs.d{,.old}
git clone https://github.com/kode9/conf-emacs.git ~/.emacs.d
emacs --debug-init &
----

CAUTION: The first start might take some time as dependencies are cloned and built.

== External dependencies

The configuration provides integration of external tools:

* https://git-scm.com/[Git^] with https://magit.vc/[Magit^]
* https://github.com/BurntSushi/ripgrep[ripgrep^] with https://github.com/minad/consult[Consult^] (default) or https://github.com/dajva/rg.el[rg.el^]
* https://github.com/ggreer/the_silver_searcher[AG the silver searcher^] with https://github.com/Wilfred/ag.el[ag.el^] (fallback)
* A spell checker such as http://aspell.net/[Aspell^] or https://hunspell.github.io/[Hunspell^] for the https://www.gnu.org/software/emacs/manual/html_node/emacs/Spelling.html[built-in spell checker^] and https://www.flycheck.org/en/latest/[Flycheck^]

== Remote Work

Two complementary workflows for remote development:

=== TRAMP (light editing)

Open remote files with `kbd:[C-x C-f] /ssh:host:/path/to/file`. TRAMP is tuned for reduced latency:

* File attribute caching (60s)
* Connection sharing via SSH ControlMaster
* Dash as TRAMP shell (lower overhead than bash)
* Lightweight magit (reduced sections over remote)
* Disabled file watchers and auto-revert for remote buffers

=== Remote Emacs (project work)

For sustained work, run an Emacs daemon on the remote machine and display its GUI locally via https://github.com/Xpra-org/xpra[XPRA^] or https://gitlab.freedesktop.org/mstoeckl/waypipe[waypipe^].

==== Requirements

.Local machine
[cols="1m,3"]
|===
|Package |Notes

|xpra |Arch: `extra`, Ubuntu: `universe`
|waypipe |Arch: `extra`, Ubuntu: `universe`. Requires local Wayland compositor.
|===

.Remote machine
[cols="1m,3"]
|===
|Package |Notes

|xpra |Ubuntu 22.04: `universe`
|waypipe |Ubuntu 22.04: `universe`. Requires Emacs built with pgtk.
|emacs |Same configuration deployed (`~/.config/emacs/`)
|dash |Ubuntu: installed by default (`/usr/bin/dash`)
|===

==== XPRA vs Waypipe

[cols="2,3,3"]
|===
|Aspect |XPRA |Waypipe

|Session persistence |Yes (survives disconnects) |No
|Remote requirements |`xpra`, X11 libs |`waypipe`, pgtk Emacs
|Local requirements |`xpra` |`waypipe`, Wayland compositor
|Protocol |Compressed frame diffs |Wayland buffer forwarding
|===

==== Troubleshooting

* **pgtk not available on remote**: waypipe requires Emacs built with pgtk. On Ubuntu 22.04, the default Emacs package uses GTK/X11. Use a PPA, Snap, or build from source with `--with-pgtk`.
* **XPRA version mismatch**: XPRA client and server versions should be compatible. Update both to the same major version.
* **Wayland not running locally**: waypipe requires a Wayland compositor on the local machine. If running X11, use XPRA instead.
* **`dash` not found on remote**: Install with `sudo apt install dash` (Ubuntu) or use `/bin/sh` as fallback.
